# Avatar Transitions API Documentation

## Overview

The transitions system provides smooth opacity-based animations between neutral avatars and emote/vowel variants. Transitions are generated by layering two complete avatars with configurable opacity values.

## Architecture

### Generation Process

1. Generate base avatar (neutral state) using `generate_deterministic(input, "neutral")`
2. Generate emote avatar (emote state) using `generate_deterministic(input, emote)`
3. Layer with opacity control:
   - Base layer opacity = 1.0 - (weight / 100)
   - Emote layer opacity = weight / 100

### Supported Transitions

**Emotes:**
- `happy` - Happy expression
- `sad` - Sad expression
- `surprised` - Surprised expression
- `angry` - Angry expression
- `bored` - Bored expression

**Vowels (for lip-sync):**
- `vowel_a` - Mouth shaped for 'A' sound
- `vowel_e` - Mouth shaped for 'E' sound
- `vowel_i` - Mouth shaped for 'I' sound
- `vowel_o` - Mouth shaped for 'O' sound
- `vowel_u` - Mouth shaped for 'U' sound

## API Endpoints

### Single-Frame Transition

**GET** `/avatar/{input}/transition/{emote}/{weight}`

Generate a single transition frame with specified blend weight.

**Parameters:**
- `input` (path, string): Email or string for deterministic generation
- `emote` (path, string): Emote name (happy, sad, etc.) or vowel (vowel_a, vowel_e, etc.)
- `weight` (path, integer): Blend weight from 0 (all base) to 100 (all emote)

**Response:**
- **200 OK**: SVG image with layered transition
- **400 Bad Request**: Invalid emote or weight out of bounds
- **500 Internal Server Error**: Generation failed

**Example:**
```bash
# Get 50% blend between neutral and happy
curl http://localhost:8000/avatar/user@example.com/transition/happy/50 > transition.svg

# Get 100% emote (fully sad)
curl http://localhost:8000/avatar/user@example.com/transition/sad/100 > sad.svg

# Get vowel transition for lip-sync
curl http://localhost:8000/avatar/user@example.com/transition/vowel_a/75 > vowel.svg
```

### Multi-Frame Bundle

**POST** `/avatar/bundle`

Generate ZIP bundle with multi-frame animation sequences.

**Request Body:**
```json
{
  "input": "user@example.com",
  "animations": ["idle", "emotes", "vowels"]
}
```

**Animation Types:**
- `idle`: 10-frame idle animation (horizontal sway + blink)
- `emotes`: 7-frame sequences for each emote (0, 25, 50, 100, 50, 25, 0)
- `vowels`: 5-frame sequences for each vowel (0, 50, 100, 50, 0)

**Emote Frame Sequence (7 frames per emote):**
```
Frame 1: weight 0   (neutral)
Frame 2: weight 25  (slight emote)
Frame 3: weight 50  (half blend)
Frame 4: weight 100 (full emote)
Frame 5: weight 50  (half blend)
Frame 6: weight 25  (slight emote)
Frame 7: weight 0   (back to neutral)
```

**Vowel Frame Sequence (5 frames per vowel):**
```
Frame 1: weight 0   (neutral/closed mouth)
Frame 2: weight 50  (half open)
Frame 3: weight 100 (fully open)
Frame 4: weight 50  (half open)
Frame 5: weight 0   (back to closed)
```

**Response:**
- **200 OK**: ZIP file containing SVG frames and metadata.json
- **400 Bad Request**: Invalid request body

**Example:**
```bash
curl -X POST http://localhost:8000/avatar/bundle \
  -H "Content-Type: application/json" \
  -d '{"input": "user@example.com", "animations": ["emotes", "vowels"]}' \
  -o bundle.zip

# Extract and view
unzip bundle.zip
ls emote_*  # List all emote frames
ls vowel_*  # List all vowel frames
cat metadata.json  # View frame metadata
```

## Bundle Metadata Structure

The `metadata.json` file included in bundles has a nested structure for transition animations:

```json
{
  "input": "user@example.com",
  "hash": "a3f2c8b4d9e1f7c2",
  "animations": {
    "emotes": {
      "frames": [
        {
          "filename": "emote_happy_weight_0.svg",
          "type": "emote",
          "emote": "happy",
          "weight": 0
        },
        {
          "filename": "emote_happy_weight_25.svg",
          "type": "emote",
          "emote": "happy",
          "weight": 25
        }
      ],
      "sequences": {
        "happy": [
          "emote_happy_weight_0.svg",
          "emote_happy_weight_25.svg",
          "emote_happy_weight_50.svg",
          "emote_happy_weight_100.svg",
          "emote_happy_weight_50.svg",
          "emote_happy_weight_25.svg",
          "emote_happy_weight_0.svg"
        ],
        "sad": ["..."]
      }
    },
    "vowels": {
      "frames": [
        {
          "filename": "vowel_a_weight_0.svg",
          "type": "vowel",
          "vowel": "a",
          "weight": 0
        }
      ],
      "sequences": {
        "a": [
          "vowel_a_weight_0.svg",
          "vowel_a_weight_50.svg",
          "vowel_a_weight_100.svg",
          "vowel_a_weight_50.svg",
          "vowel_a_weight_0.svg"
        ],
        "e": ["..."]
      }
    }
  }
}
```

**Key Structure Points:**
- `metadata["animations"]["emotes"]["frames"]` - Array of all unique emote frame objects
- `metadata["animations"]["emotes"]["sequences"]` - Object mapping emote names to playback order arrays
- `metadata["animations"]["vowels"]["frames"]` - Array of all unique vowel frame objects
- `metadata["animations"]["vowels"]["sequences"]` - Object mapping vowel letters to playback order arrays

## Caching

All transitions are cached to the filesystem for performance:

**Cache Location:** `out/avatar/`

**Cache Key Format:**
```
{hash16}_transition_{emote}_{weight}.svg
```

Where `hash16` is the first 16 characters of SHA-256 hash of input string.

**Example Cache Files:**
```
a3f2c8b4d9e1f7c2_transition_happy_50.svg
a3f2c8b4d9e1f7c2_transition_sad_100.svg
a3f2c8b4d9e1f7c2_transition_vowel_a_75.svg
```

**Cache Invalidation:**
To regenerate transitions (e.g., after asset updates), delete cache files:
```bash
rm -rf out/avatar/*_transition_*
```

## Usage Examples

### Animation Timing

**Emote Animation (7 frames @ 300ms = 2.1 seconds):**
```javascript
const emoteFrames = [0, 25, 50, 100, 50, 25, 0];
const FRAME_DURATION = 300; // milliseconds

let frameIndex = 0;
setInterval(() => {
    const weight = emoteFrames[frameIndex];
    const url = `/avatar/user@example.com/transition/happy/${weight}`;
    avatarImg.src = url;
    frameIndex = (frameIndex + 1) % emoteFrames.length;
}, FRAME_DURATION);
```

**Vowel Animation (5 frames @ 300ms = 1.5 seconds):**
```javascript
const vowelFrames = [0, 50, 100, 50, 0];
const FRAME_DURATION = 300;

let frameIndex = 0;
setInterval(() => {
    const weight = vowelFrames[frameIndex];
    const url = `/avatar/user@example.com/transition/vowel_a/${weight}`;
    avatarImg.src = url;
    frameIndex = (frameIndex + 1) % vowelFrames.length;
}, FRAME_DURATION);
```

### Lip-Sync Example

Map text to vowel animations:

```javascript
function getVowelForLetter(letter) {
    const vowels = {
        'a': 'vowel_a', 'e': 'vowel_e', 'i': 'vowel_i',
        'o': 'vowel_o', 'u': 'vowel_u'
    };
    return vowels[letter.toLowerCase()] || null;
}

async function animateSpeech(text, input) {
    const letters = text.toLowerCase().split('');

    for (const letter of letters) {
        const vowel = getVowelForLetter(letter);
        if (vowel) {
            // Play vowel animation (0 -> 100 -> 0)
            for (const weight of [0, 50, 100, 50, 0]) {
                avatarImg.src = `/avatar/${input}/transition/${vowel}/${weight}`;
                await sleep(300);
            }
        } else {
            // Hold neutral for consonants
            avatarImg.src = `/avatar/${input}/transition/vowel_a/0`;
            await sleep(150);
        }
    }
}
```

### Using Bundle Metadata

Load and play animations from a downloaded bundle:

```javascript
// After extracting bundle.zip
async function loadBundle() {
    const response = await fetch('metadata.json');
    const metadata = await response.json();

    // Play a specific emote sequence
    const happySequence = metadata.animations.emotes.sequences.happy;
    for (const filename of happySequence) {
        avatarImg.src = filename;
        await sleep(300);
    }

    // Play a specific vowel sequence
    const aSequence = metadata.animations.vowels.sequences.a;
    for (const filename of aSequence) {
        avatarImg.src = filename;
        await sleep(300);
    }
}
```

### React Component Example

```jsx
import { useState, useEffect } from 'react';

function AvatarTransition({ input, emote, autoPlay = false }) {
    const [weight, setWeight] = useState(0);
    const [playing, setPlaying] = useState(false);

    const frames = [0, 25, 50, 100, 50, 25, 0];

    useEffect(() => {
        if (!playing) return;

        let frameIndex = 0;
        const interval = setInterval(() => {
            setWeight(frames[frameIndex]);
            frameIndex = (frameIndex + 1) % frames.length;
        }, 300);

        return () => clearInterval(interval);
    }, [playing]);

    return (
        <div>
            <img
                src={`/avatar/${input}/transition/${emote}/${weight}`}
                alt="Avatar"
            />
            <input
                type="range"
                min="0"
                max="100"
                value={weight}
                onChange={(e) => setWeight(e.target.value)}
                disabled={playing}
            />
            <button onClick={() => setPlaying(!playing)}>
                {playing ? 'Stop' : 'Play'}
            </button>
        </div>
    );
}
```

## Error Handling

**Invalid Emote (400):**
```json
{
  "detail": "Unknown emote: invalid_emote. Valid options: ['happy', 'sad', 'surprised', 'angry', 'bored', 'vowel_a', 'vowel_e', 'vowel_i', 'vowel_o', 'vowel_u']"
}
```

**Invalid Weight (400):**
```json
{
  "detail": "Weight must be between 0 and 100, got 150"
}
```

**Internal Error (500):**
```json
{
  "detail": "Internal server error"
}
```

## Performance

**Generation Time:**
- First request (cold): ~50-200ms (generates and caches)
- Cached requests: ~5-20ms (serves from cache)

**Bundle Generation:**
- Emotes only (5 emotes × 4 unique weights): ~20 unique SVG files
- Vowels only (5 vowels × 3 unique weights): ~15 unique SVG files
- All animations: ~35 unique SVG files + 10 idle frames

**Bundle Optimization:**
The bundle system is optimized to only generate unique weight values:
- Emote sequence `[0, 25, 50, 100, 50, 25, 0]` only generates 4 files (0, 25, 50, 100)
- Vowel sequence `[0, 50, 100, 50, 0]` only generates 3 files (0, 50, 100)
- The `sequences` object maps which files to play in what order

**Cache Size:**
- Per transition frame: ~5-15 KB (SVG)
- Full bundle (all animations): ~400-600 KB
