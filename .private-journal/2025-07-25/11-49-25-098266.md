## Project Notes

Successfully unified the hair positioning system by combining data-offset-y and data-position-y into a single attribute. The new system supports both special cases (like "between-body-eyes") and percentage-based positioning (like "25%" for 25% down from body top). This makes the API more consistent and intuitive - similar to how data-position-x works with its special cases.

Updated all components:

- parse_defs() function to remove offset_y parameter
- agent_svg() positioning logic to handle percentage parsing
- Both hair SVG files to use the new unified attributes
- Documentation in README.md and CLAUDE.md

The percentage parsing uses simple string detection (endswith("%")) and float conversion, which provides flexible positioning while keeping the implementation straightforward.
