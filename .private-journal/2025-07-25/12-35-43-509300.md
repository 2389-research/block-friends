## Project Notes

Fixed a critical positioning inconsistency in the hair system. The issue was that percentage-based positioning was using body dimensions for the base calculation but hair dimensions for anchor offsets, causing different-sized hair assets to appear at inconsistent positions even with the same percentage values.

**The Problem:**

- X positioning: `hair_x = PAD + (BOX * percent) - hair_w / 2` (used cell width)
- Y positioning: `base_y = by0 + (body_h * percent)` (used body height)
- But anchor offsets used hair dimensions: `hair_y = base_y - hair_h`

This meant a hair asset with height 14px and one with height 7px would end up at different final positions even with the same percentage, because the body-relative base position was the same but the hair-relative anchor offset was different.

**The Fix:**
Changed percentage calculations to be relative to hair dimensions:

- X positioning: `hair_x = cx + (hair_w * percent) - hair_w / 2` (60% = 60% of hair width from body center)
- Y positioning: `base_y = by0 + (hair_h * percent)` (-6% = -6% of hair height from body top)

This creates consistent positioning behavior where percentage values have the same visual meaning regardless of hair asset size.
