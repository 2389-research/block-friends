## Project Notes

Successfully unified both positioning axes (X and Y) to support the same pattern: special cases and percentage values. Now both data-position-x and data-position-y work consistently:

- Special cases: "body-center", "cell-center", "above-body", "between-body-eyes"  
- Percentage values: "25%", "50%", "75%" etc. for precise positioning

The implementation uses the same pattern for both axes - checking for endswith("%") to distinguish percentage values from special cases. This creates a consistent, intuitive API that's easy to document and use.

No changes were needed to existing hair SVG files since they already use special cases that remain supported.
